Список задач

В данной форме расположен общий список задач по всем проектам и сотрудникам, отсортированный по заданному приоритету.
  • Особенности использования
  • Основная панель
  • Панель быстрого доступа к комментариям

Особенности использования

  • Предлагаемая схема работы:
    • Руководитель отдела разработки ранжирует задачи по приоритету и назначает исполнителей, исходя из аналитической информации, представленной в списке (крайний срок задачи, обещанная оценка, загруженность исполнителя).
    • Разработчик назначает на себя свободные задачи, видит пул задач других разработчиков. 
    • Аудитор отслеживает состояние задач, плановые сроки их выполнения, фактически затраченное время на выполнение, а также может добавить задачу в  список контролируемых задач.
  • Команды контекстного меню
    • "Задача важна! На контроль! / Задача уже не важна :(" - позволяет управлять списком контролируемых задач текущего пользователя.
      Команда доступна для пользователей с ролями Полные права и Наблюдатель.
    • Список исполнителей - позволяет быстро назначить для задачи исполнителя.
      Команда доступна для пользователей с ролями Полные права, Опытный разработчик, Разработчик.
           Пользователи с ролями Полные права,  Опытный разработчик могут назначать на задачу любого исполнителя.
           Пользователи с ролью Разработчик могут назначать задачи только на себя.
    • "Перенести в пожелания" - позваляет перенести задачу из списка оперативных задач в список пожеланий.

    1. Список задач
      Общий список задач, отсортированный по заданному приоритету.

      Колонка Описание Ограничения доступа
      N (приоритет задачи)

      Глобальный приоритет задачи среди всех задач в списке.
       Новые задачи, у которых не назначен приоритет, отображаются вверху списка в порядке их появления, с пустым приоритетом.

      		 Доступна для редактирования пользователям с ролью Полные права.
      V (приоритет аудиторов задачи) Показывает приоритет задачи среди аудиторов.
      Рассчитывается как среднее арифметическое приоритетов среди всех аудиторов задачи.      		 Только просмотр.
      Задача  Наименование задачи. Только просмотр.
       Все пользователи системы видят полный список задач вне зависимости от роли.
      Проект Проект, к которому относится задача. Доступна для редактирования пользователям с ролями: Полные права, Опытный разработчик, Разработчик
      Исполнитель Исполнитель, на которого назначена задача.

      Пользователи с ролями Полные права,  Опытный разработчик могут назначать на задачу любого исполнителя.
      Пользователи с ролью Разработчик могут назначать задачи только на себя.

      Дата добавления Дата добавления задачи. Только просмотр
       Обещанная оценка Первичная оценка времени выполнения задачи (в часах), данная аудитору. Видима и доступна для редактирования только пользователям с ролью Полные права.
       Крайний срок Срок, к которому необходимо выполнить задачу.
      Равен максимальному сроку, заданному аудиторами задачи или пользователями с ролью Полные права.      		 Видима и доступна для редактирования только пользователям с ролью Полные права.
       Текущая оценка Текущая (уточнённая) оценка времени выполнения задачи (в часах) Пользователи с ролями Полные права, Опытный разработчик могут редактировать оценку любого исполнителя.
      Пользователи с ролью Разработчик могут редактировать оценку только своих задач.
       Потрачено времени Фактически затраченное время на выполнение задачи (в часах). Только просмотр.
       Осталось времени Разница между текущей оценкой и фактически затраченным временем (в часах). Только просмотр.
       Плановый срок

      Плановая дата завершения задачи исполнителем.
      Рассчитывается исходя из плана задач исполнителя и рабочего календаря.

      		 Только просмотр.
      Статус Текущий статус задачи. Пользователи с ролями Полные права, Опытный разработчик могут редактировать статус любого исполнителя.
      Пользователи с ролью Разработчик могут редактировать статус только своих задач.

    2. Панель действий

       Создать - добавление новой задачи в список; 

              Добавление задачи осуществляется на той же форме, с помощью поля ввода новой задачи:

            

       - открыть форму текущей задачи;

       - отбор по значению текущей колонки;

       - отменить отбор;

       - расширенная форма настройки списка;

       - показать панель быстрого отбора по статусу;

       - показать панель быстрого доступа к комментариям;
